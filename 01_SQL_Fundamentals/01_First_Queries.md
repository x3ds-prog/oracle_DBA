# Модуль 1: Основы SQL

## Урок 1: Ваши первые запросы

Добро пожаловать в мир SQL (Structured Query Language)! Если вы привыкли к императивным языкам или скриптам в bash, где вы описываете *шаги* для достижения результата, то SQL поначалу может показаться странным.

SQL — это **декларативный** язык. Вы не говорите базе данных, *как* сделать что-то, вы описываете, *какой результат* вы хотите получить. А база данных сама решает, как наиболее эффективно выполнить ваш запрос.

### Ваш первый запрос: Таблица `DUAL`

В Oracle есть специальная встроенная таблица `DUAL`, которая всегда содержит ровно одну строку и одну колонку. Она идеально подходит для "разогрева" и выполнения запросов, которые не требуют данных из реальных таблиц. Это ваш аналог команды `echo` для проверки простых выражений.

Подключитесь к вашей базе данных через `sqlplus` или `DBeaver` и попробуйте выполнить этот запрос:

```sql
SELECT 100 + 50 FROM dual;
```

Результат будет `150`. Все просто. А теперь давайте познакомимся с главной особенностью баз данных.

### Особый случай: `NULL`

В мире Linux пустая переменная — это просто пустая строка. В SQL есть специальное значение `NULL`, которое означает "неизвестность" или "отсутствие данных". Это не ноль и не пустая строка. Это фундаментальное понятие.

Посмотрите, как `NULL` ведет себя в арифметике.

```sql
SELECT 100 + 50 + NULL FROM dual;
```

Результатом будет не `150`, а `NULL`.

> **Основное правило:** Любая арифметическая операция с `NULL` всегда возвращает `NULL`.

С конкатенацией (объединением) строк в Oracle все немного иначе и контринтуитивно для программиста:

```sql
SELECT 'Hello' || NULL || ' World' FROM dual;
```
*Оператор `||` в Oracle - это стандартный способ конкатенации строк.*

Результатом будет `Hello World`.

> **Особенность Oracle:** При конкатенации строк Oracle обрабатывает `NULL` как пустую строку. В других СУБД (например, PostgreSQL или SQL Server) результат был бы `NULL`. Запомните это поведение, это частый вопрос на собеседованиях.

### Создание вашей первой таблицы

Давайте перейдем от `dual` к реальным данным. Нам нужна таблица для экспериментов. В SQL структура данных (схема) так же важна, как и сами данные.

Команда `CREATE TABLE` определяет "колонки" нашей таблицы и типы данных, которые они могут хранить.

```sql
-- Создаем простую таблицу для хранения данных о сотрудниках
CREATE TABLE employees (
    id NUMBER,
    name VARCHAR2(50),
    salary NUMBER
);
```
> **Для админа Linux: Аналогии**
>
> -   `CREATE TABLE`: Это как определить структуру для CSV файла, задав имена колонок в первой строке: `id,name,salary`.
> -   `NUMBER`, `VARCHAR2(50)`: Это типы данных. `NUMBER` для чисел (целых и дробных), `VARCHAR2(50)` для строк с максимальной длиной 50 символов. Это как проверка типов в скриптах, чтобы в поле для зарплаты не попало имя.

### Добавление данных: `INSERT`

Теперь, когда у нас есть структура, давайте добавим в нее данные с помощью команды `INSERT INTO`.

```sql
-- Добавляем первого сотрудника
INSERT INTO employees VALUES (1, 'John Doe', 50000);

-- Добавляем второго сотрудника
INSERT INTO employees VALUES (2, 'Jane Smith', 60000);
```

### Извлечение данных: `SELECT`

Команда `SELECT` — это ваш основной инструмент для просмотра данных.

-   `SELECT *`: означает "показать все колонки".
-   `FROM employees`: указывает, из какой таблицы брать данные.

```sql
SELECT * FROM employees;
```

**Результат:**
```
        ID NAME                                                   SALARY
---------- -------------------------------------------------- ----------
         1 John Doe                                                50000
         2 Jane Smith                                              60000
```

### Что дальше?

Вы научились создавать таблицы, вставлять в них данные и делать простейшие запросы. В следующем уроке мы разберем, как фильтровать данные с помощью `WHERE` и поймем, в каком порядке Oracle на самом деле выполняет ваш запрос.
